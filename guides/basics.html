// VARIABLES & TYPES

Variables & Type System

Go is statically typed. Every variable has a type known at compile time. Go infers types when possible — keeping code concise without sacrificing safety.

declaration styles
// Explicit type
var name string = "Gopher"
var age  int    = 10

// Type inferred
var lang = "Go"

// Short declaration (inside func only)
city := "Chennai"
count := 42

// Multiple vars
x, y := 10, 20

// Zero values — no undefined!
var n   int     // 0
var s   string  // ""
var ok  bool    // false
constants & iota
const Pi = 3.14159
const MaxRetries = 3

// iota — auto-incrementing constants
type Direction int

const (
    North Direction = iota // 0
    South                   // 1
    East                    // 2
    West                    // 3
)

// Bit flags with iota
const (
    Read  = 1 << iota // 1
    Write               // 2
    Exec                // 4
)

// FUNCTIONS

Functions — First Class Citizens

Go functions are clean, explicit, and support multiple return values — which is how idiomatic Go handles errors.

functions.go
// Basic function
func add(a, b int) int {
    return a + b
}

// Multiple return values — idiomatic Go
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("division by zero")
    }
    return a / b, nil
}

// Variadic function
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

// First-class: functions as values
func apply(f func(int) int, v int) int {
    return f(v)
}

// Closure
func counter() func() int {
    n := 0
    return func() int {
        n++
        return n
    }
}
// STRUCTS & INTERFACES

Structs, Methods & Interfaces

Go has no classes. Instead: structs with methods, and interfaces satisfied implicitly — one of Go's most elegant design choices.

structs_interfaces.go
// Define a struct
type Animal struct {
    Name  string
    Sound string
    Legs  int
}

// Method on a struct (pointer receiver — can mutate)
func (a *Animal) Speak() string {
    return fmt.Sprintf("%s says %s", a.Name, a.Sound)
}

// Interface — implicit satisfaction
type Speaker interface {
    Speak() string
}

// Animal implements Speaker automatically — no 'implements' keyword
func makeNoise(s Speaker) {
    fmt.Println(s.Speak())
}

func main() {
    dog := &Animal{Name: "Dog", Sound: "Woof", Legs: 4}
    makeNoise(dog) // "Dog says Woof"
}
◈
Interface Satisfaction is Implicit In Go, a type satisfies an interface simply by implementing its methods. No explicit declaration like implements Speaker is needed. This is called structural typing and is one of Go's most powerful features — it decouples your code naturally.
// ERROR HANDLING

Errors as Values

Go has no exceptions. Errors are plain values returned from functions. This forces you to handle them explicitly — which leads to more reliable, predictable code.

errors.go — idiomatic patterns
// Sentinel errors — compare with ==
var ErrNotFound = errors.New("not found")

// Custom error type with context
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation error on %s: %s", e.Field, e.Message)
}

// Wrapping errors (Go 1.13+)
func loadUser(id int) (*User, error) {
    u, err := db.FindUser(id)
    if err != nil {
        return nil, fmt.Errorf("loadUser %d: %w", id, err)
    }
    return u, nil
}

// Unwrapping to check type
if errors.Is(err, ErrNotFound) { ... }
var valErr *ValidationError
if errors.As(err, &valErr) { ... }
✓
Always wrap errors with context Use fmt.Errorf("doing X: %w", err) to add context as errors bubble up. This builds a clear error chain that makes debugging production issues far easier. The %w verb preserves the original error for errors.Is and errors.As.
