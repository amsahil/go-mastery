Go's standard library ships a production-grade HTTP server out of the box. No framework required. Build REST APIs, middleware chains, JSON services, and websockets — all with net/http.

// HTTP VERBS

REST Methods at a Glance

Each HTTP method has a specific semantic contract. Violating these leads to broken caches, bad clients, and confused consumers.

GET

Read resource

Safe & idempotent. Never mutates state. Can be cached freely.

POST

Create resource

Not idempotent. Creates a new entity. Returns 201 Created + Location header.

PUT

Replace resource

Idempotent full replacement. Send entire resource body. Returns 200 or 204.

PATCH

Partial update

Send only fields to change. Not guaranteed idempotent. Returns 200 or 204.

DELETE

Remove resource

Idempotent. Deleting something twice should not error. Returns 204 No Content.

Building Your First HTTP Server

Go 1.22 upgraded net/http with method-based and wildcard routing built in — no third-party router needed for most use cases.

server.go — method routing (Go 1.22+)GO
package main

import (
    "encoding/json"
    "log/slog"
    "net/http"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func main() {
    mux := http.NewServeMux()

    // Method + path pattern (Go 1.22)
    mux.HandleFunc("GET /users", listUsers)
    mux.HandleFunc("POST /users", createUser)
    mux.HandleFunc("GET /users/{id}", getUser)
    mux.HandleFunc("DELETE /users/{id}", deleteUser)

    slog.Info("server started", "addr", ":8080")
    http.ListenAndServe(":8080", mux)
}

func getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id") // extract {id} from path
    user := User{ID: 1, Name: "Alice"}
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(user)
    _ = id
}
✓
Go 1.22 routing is genuinely good The new ServeMux supports method routing, path wildcards, and precedence rules. For most CRUD APIs you no longer need Gorilla Mux, Chi, or Gin just for routing.
// JSON

JSON Encoding & Decoding

Go's encoding/json package handles serialization with struct tags. Clean, explicit, and zero-dependency.

decode request body
func createUser(
    w http.ResponseWriter,
    r *http.Request,
) {
    var input struct {
        Name  string `json:"name"`
        Email string `json:"email"`
    }

    dec := json.NewDecoder(r.Body)
    dec.DisallowUnknownFields()
    if err := dec.Decode(&input); err != nil {
        http.Error(w, err.Error(), 400)
        return
    }

    // input.Name and input.Email are safe
    w.WriteHeader(http.StatusCreated)
}
struct tags reference
type Product struct {
    // Field → JSON key mapping
    ID    int    `json:"id"`
    Name  string `json:"name"`

    // Omit if empty
    Desc  string `json:"desc,omitempty"`

    // Never include in JSON
    Secret string `json:"-"`

    // Pointer: null-able in JSON
    Price *float64 `json:"price"`

    // Nested struct
    Meta Metadata `json:"meta"`
}
// MIDDLEWARE

Middleware Chains

Go middleware is just a function that wraps an http.Handler. Chain them together to build request pipelines.

→ Request In → Incoming HTTP request arrives at server
Logger → Log method, path, duration, status code
Auth → Validate JWT / API key — return 401 if invalid
Rate Limiter → Throttle per-IP requests — return 429 if exceeded
Handler → Your actual business logic executes here
← Response Out Middleware can also post-process the response
middleware.go — logger + auth pattern
// Middleware signature: wrap an http.Handler, return http.Handler
func Logger(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        slog.Info("request",
            "method", r.Method,
            "path",   r.URL.Path,
            "dur",    time.Since(start),
        )
    })
}

func RequireAuth(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header().Get("Authorization")
        if !isValidToken(token) {
            http.Error(w, "unauthorized", http.StatusUnauthorized)
            return
        }
        next.ServeHTTP(w, r)
    })
}

// Stack middleware: outermost runs first
handler := Logger(RequireAuth(mux))
// HTTP CLIENT

Making HTTP Requests

Go's http.Client is powerful and production-ready — always configure timeouts, always close response bodies.

client.go — production HTTP client
// Configure a reusable client with timeout
var client = &http.Client{
    Timeout: 10 * time.Second,
    Transport: &http.Transport{
        MaxIdleConns:       100,
        IdleConnTimeout:    90 * time.Second,
        DisableCompression: false,
    },
}

func getJSON(ctx context.Context, url string, out any) error {
    req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
    if err != nil {
        return fmt.Errorf("build request: %w", err)
    }
    req.Header.Set("Accept", "application/json")

    resp, err := client.Do(req)
    if err != nil {
        return err
    }
    defer resp.Body.Close() // ALWAYS close body

    if resp.StatusCode != http.StatusOK {
        return fmt.Errorf("unexpected status: %d", resp.StatusCode)
    }

    return json.NewDecoder(resp.Body).Decode(out)
}
⚠
Never use http.DefaultClient in production http.DefaultClient has no timeout. A single slow server can hang your goroutine forever. Always create a custom http.Client with an explicit Timeout field.

// PRODUCTION

Graceful Shutdown

Never kill your server abruptly. Use http.Server.Shutdown to drain in-flight requests before exit.

graceful_shutdown.go
func main() {
    srv := &http.Server{Addr: ":8080", Handler: mux}

    // Start server in goroutine
    go func() {
        if err := srv.ListenAndServe(); err != http.ErrServerClosed {
            log.Fatal(err)
        }
    }()

    // Wait for interrupt signal
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit

    // Give in-flight requests 30s to finish
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    if err := srv.Shutdown(ctx); err != nil {
        log.Fatal("shutdown error:", err)
    }
    slog.Info("server stopped cleanly")
}
