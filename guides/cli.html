CLI Tools &
Tooling

Go is the language of CLI tools — Docker, Kubernetes, Terraform, and Hugo are all Go CLIs. Build your own: parse flags, read stdin, write structured output, and ship a single binary anywhere.

// GO TOOLCHAIN

The go Command — Your Swiss Army Knife

Every Go developer lives in the terminal. These are the commands you'll use every single day.

go run
go run main.go

Compile and run in one step. No binary left on disk. Great for scripts and quick iteration.

go build
go build -o ./bin/app .

Compile to a static binary. Combine with GOOS/GOARCH env vars to cross-compile for any platform.

go test
go test ./... -race

Run all tests, recursively. -race enables the data race detector — always use it in CI.

go mod tidy
go mod tidy

Add missing and remove unused dependencies from go.mod and go.sum. Run before every commit.

go vet
go vet ./...

Static analysis for common mistakes: unreachable code, bad printf formats, suspicious constructs.

go generate
go generate ./...

Run code generation tools declared via //go:generate comments. Used for mocks, protobuf, and stringer.

Makefile — standard Go project tasks
# Build for current platform
build:
	go build -ldflags="-s -w" -o bin/app .

# Cross-compile for Linux amd64
build-linux:
	GOOS=linux GOARCH=amd64 go build -o bin/app-linux .

# Run tests with race detector and coverage
test:
	go test -race -coverprofile=coverage.out ./...
	go tool cover -html=coverage.out

# Format, vet, and lint
lint:
	gofmt -w .
	go vet ./...
	staticcheck ./...

# Clean build artifacts
clean:
	rm -rf bin/ coverage.out
// FLAG PACKAGE

Parsing Flags with flag

Go's built-in flag package handles CLI flags cleanly. For complex CLIs with subcommands, use cobra.

main.go — flag parsing
package main

import (
    "flag"
    "fmt"
    "os"
)

func main() {
    // Define flags with name, default, and help text
    host    := flag.String("host", "localhost", "server hostname")
    port    := flag.Int("port", 8080, "server port")
    verbose := flag.Bool("verbose", false, "enable verbose logging")
    output  := flag.String("output", "json", "output format: json|text|csv")

    flag.Parse() // Parse os.Args[1:]

    // Positional args after flags
    args := flag.Args()

    if *verbose {
        fmt.Printf("Connecting to %s:%d (format: %s)\n", *host, *port, *output)
    }

    if len(args) == 0 {
        fmt.Fprintln(os.Stderr, "error: no command specified")
        flag.Usage()
        os.Exit(1)
    }
}
// Usage: ./app -host=api.example.com -port=443 -verbose get /users
Flag FunctionTypeExample
flag.String()*string-output=json
flag.Int()*int-port=8080
flag.Bool()*bool-verbose (presence = true)
flag.Float64()*float64-rate=1.5
flag.Duration()*time.Duration-timeout=30s
flag.StringVar()writes to varflag.StringVar(&host, "host", ...)
// COBRA

Subcommands with Cobra

For CLIs with multiple subcommands (like git commit, docker run), Cobra is the de-facto standard.

cmd/root.go — cobra CLI structure
var rootCmd = &cobra.Command{
    Use:   "myapp",
    Short: "A Go CLI application",
    Long:  `myapp is a CLI tool for doing awesome things.`,
}

var serveCmd = &cobra.Command{
    Use:   "serve",
    Short: "Start the HTTP server",
    RunE: func(cmd *cobra.Command, args []string) error {
        port, _ := cmd.Flags().GetInt("port")
        return startServer(port)
    },
}

func init() {
    serveCmd.Flags().IntP("port", "p", 8080, "port to listen on")
    rootCmd.AddCommand(serveCmd)
}

// Usage: myapp serve --port 9090
// myapp help serve
◈
Recommended CLI project layout Put each subcommand in its own file under cmd/: cmd/root.go, cmd/serve.go, cmd/migrate.go. Keep business logic in internal/ — not in the command files themselves.
// STDIN & STDOUT

Reading Stdin, Writing Output

Unix pipelines make CLIs composable. Read from stdin, write to stdout, errors to stderr — always.

read stdin line by line
func main() {
    scanner := bufio.NewScanner(os.Stdin)
    for scanner.Scan() {
        line := scanner.Text()
        // process line
        fmt.Println(line)
    }
    if err := scanner.Err(); err != nil {
        fmt.Fprintln(os.Stderr, err)
        os.Exit(1)
    }
}
structured JSON output
type Result struct {
    Status  string `json:"status"`
    Message string `json:"message"`
}

enc := json.NewEncoder(os.Stdout)
enc.SetIndent("", "  ")
enc.Encode(Result{
    Status:  "ok",
    Message: "done",
})
// Pipe: ./app | jq '.status'
// TESTING

Testing in Go

Go ships a first-class testing framework in the standard library. No assertion library needed — though testify is popular for readability.

math_test.go — table-driven tests
func TestDivide(t *testing.T) {
    tests := []struct{
        name    string
        a, b    float64
        want    float64
        wantErr bool
    }{
        {"positive", 10, 2, 5, false},
        {"negative", -9, 3, -3, false},
        {"div by zero", 5, 0, 0, true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got, err := Divide(tt.a, tt.b)
            if (err != nil) != tt.wantErr {
                t.Fatalf("err=%v, wantErr=%v", err, tt.wantErr)
            }
            if got != tt.want {
                t.Errorf("got %v, want %v", got, tt.want)
            }
        })
    }
}
bash — test commands
# Run all tests
go test ./...

# Run with race detector (always in CI)
go test -race ./...

# Run a specific test
go test -run TestDivide ./...

# Show coverage in browser
go test -coverprofile=cover.out ./...
go tool cover -html=cover.out

# Benchmark
go test -bench=. -benchmem ./...

# Verbose output
go test -v ./...
✓
Always use table-driven tests Table-driven tests are the idiomatic Go testing pattern. They make adding new cases trivial, produce clear failure messages with t.Run(name, ...), and keep your test file lean.
